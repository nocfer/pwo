PWO Directions

Design Direction Exploration

Six layout approaches for the PWO workout execution screen, all using the same dark-first design system with DM Sans, tuned indigo, and minimalistic aesthetics.

What stays the same across all directions

Color palette, typography, spacing scale, 48pt touch targets, set indicators, rest timer persistence, non-linear navigation support, and pre-fill logic. The difference is how the exercise list and set logging area are arranged on screen.

Strengths

  • Full workout overview always visible — scroll to see all exercises
  • Tap any exercise or set dot for instant navigation
  • Minimal cognitive load — familiar list pattern
  • Set dots provide at-a-glance progress for every exercise
  • Compact rows maximize exercises visible without scroll

Trade-offs

  • Scroll position shifts when expanding/collapsing

Implementation notes

Evolves the existing WorkoutMatrix.tsx component. Add expand/collapse animation (LayoutAnimation or Reanimated). Set dots become tappable navigation targets. Scroll-to behavior centers expanded exercise.

Recommendation

Direction A (Matrix Accordion) is the strongest fit for PWO. It builds directly on the existing WorkoutMatrix, supports fully arbitrary navigation through tappable set dots and exercise rows, maintains a scrollable overview of the entire workout, and achieves the minimalist aesthetic with the least visual chrome. It's also the simplest to implement — an evolution of existing code rather than a new pattern.

Direction E (Split Matrix) is the strongest alternative if you want the overview to be always visible without scrolling, at the cost of reduced detail area height and more complex layout management.
